🔧 Update Hub

Push UI or data patches without touching your progress. Your chapter ticks are always safe in localStorage.

How data safety works
💾
Your chapter progress is stored in localStorage under the key cma-progress-v4. This key never changes across app updates.
🔄
When you update index.html on GitHub, only the UI code changes. localStorage stays untouched — your ticks survive every update.
📱
Data lives on your phone's Chrome for the GitHub Pages URL. As long as you don't clear Chrome data, it's permanent. Export below as backup.
⚡
Use the patch box to update planner data, chapter lists, or law targets without getting a new file from me — just paste and push.
UI / Data patch
Patch runs in a sandboxed function with access to RUNTIME and api. Chapter IDs must stay stable — if you rename an ID, use api.remapProgress({'old-id':'new-id'}) to preserve ticks.
Data backup & restore
Export saves a cma-progress.json file you can keep as backup or use to restore on another device/browser. Import loads it back instantly.
